# NoirStock PWA v3

iPhone向け、黒基調、オフライン前提の在庫・販売・経費・帳簿管理PWAです。

## この版で追加したもの
- 前年繰越ウィザード
- 販売証跡画像の保存
- 月別推移チャート
- フォーム下書き自動保存
- 重複候補を含む要確認リスト強化

## この版でできること
- 在庫登録 / 編集 / 削除
- 商品写真の端末内保存
- 販売結果入力 / 編集 / 削除
- 販売証跡画像の保存
- メルカリ取引画面テキストの貼り付け解析
- 共通経費入力 / 編集 / 削除
- 年間サマリー / 月別集計 / 利益上位商品
- JSONバックアップ / 復元
- 前年バックアップからの在庫繰越
- 在庫 / 売上 / 経費 / 帳簿CSV出力
- Service Worker + IndexedDB によるオフライン対応

## この版でまだ入れていないもの
- 画像OCRエンジンそのもの
- カメラからの自動文字認識
- Excel直接出力

## iPhoneで使う手順
1. このフォルダを静的ホスティングに置く
2. iPhone の Safari で `index.html` を開く
3. 共有メニューから **ホーム画面に追加**
4. 初回起動後、設定で JSON バックアップを一度試す

## ローカルで試す手順
```bash
cd noirstock_pwa_v3
python3 -m http.server 8000
```
ブラウザで `http://localhost:8000` を開きます。

## 利益計算ルール
- 実受取額 = 商品代金 - 販売手数料 - 送料（未入力時自動）
- 売上原価 = 仕入単価 × 販売数量
- 粗利益 = 実受取額 - 売上原価
- 実利益 = 粗利益 - 商品別追加経費
- 年間サマリーの実利益 = 販売実利益合計 - 共通経費

## 前年繰越の考え方
- 前年の仕入を当年の仕入に二重計上しません
- 前年末に残っていた数量だけ、新規在庫として作成します
- 作成された在庫には `前年繰越` 年が入ります

## バックアップの注意
- データは端末内の IndexedDB に保存されます
- Safari のサイトデータ削除や端末変更で消える可能性があります
- 定期的に JSON バックアップを書き出してください


## v4で追加したもの

- 既存帳簿から作成した**移行JSONの取り込みUI**
- **置換取り込み** と **追加取り込み** の2モード
- 取り込み前の**件数プレビュー**

## 既存帳簿データを反映する手順

1. 設定を開く
2. **既存帳簿の移行JSONを読み込む** を押す
3. `noirstock_migration_2025_2026.json` を選ぶ
4. 空の状態なら **現在データを置換して移行** を押す
5. すでに入力済みデータを残したい場合だけ **現在データへ追加して移行** を使う
